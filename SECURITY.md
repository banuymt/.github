# Security Policy

Guvenlik aciklarini sorumlu sekilde paylasin.

## Raporlama

- E-posta: `info@yazilimmuhendisligi.com.tr`
- Konu: `[SECURITY] <kisa-baslik>`
- Icerik: etkisi, tekrar adimlari, PoC (varsa), onerilen cozum

## Kapsam

- Yetkilendirme/acik erisim problemleri
- Kimlik bilgisi sizmasi
- Enjeksiyon, XSS, CSRF, SSRF vb.
- CI/CD veya altyapi guvenlik riskleri

Raporlar gizli ele alinir ve dogrulandiginda hizli sekilde aksiyon planlanir.
